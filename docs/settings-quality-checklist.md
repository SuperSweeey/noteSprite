# 设置系统质量清单

设置页是信任入口。任何设置类功能都必须避免“看起来保存了、实际没生效”“刷新后回退”“保存时污染数据”。

## 高风险 Bug 类型

1. 前端状态生效，但没有写入后端
   - 表现：点击后当前页面变了，刷新后恢复旧值。
   - 防线：所有长期设置必须经过 `/api/settings` PUT，并有真实 GET/PUT/GET 回归。

2. 后端保存了，但业务调用没有读取
   - 表现：设置页显示已改，AI 对话、解读、字体、转录仍使用旧配置。
   - 防线：设置项必须进入 `resolveSettings`，并由调用方读取统一配置。

3. 局部更新覆盖其他字段
   - 表现：只改一个开关，其他布尔值、Prompt、密钥变空或变默认。
   - 防线：PUT 合并必须使用 `??` 和字段级合并，不允许用 `Boolean(...)` 或整对象覆盖。

4. 密钥遮罩覆盖真实密钥
   - 表现：页面显示 `••••xxxx`，保存后真实 key 被遮罩字符串替代。
   - 防线：保存时识别遮罩值，保留 existing secret。

5. 中文 Prompt 编码污染
   - 表现：出现 `ä½ æ¯...`、`ï¿½`、`�`、`å/æ/ç` 等乱码；点击默认按钮又恢复。
   - 防线：Prompt 读取和保存都必须检测 mojibake；发现污染时回退默认或保留干净 existing。

6. 静态缓存导致设置读旧值
   - 表现：API 或页面似乎不是最新配置。
   - 防线：设置 API 和依赖设置的根布局必须 `force-dynamic`。

7. `npm run build` 后 dev server 静态资源错位
   - 表现：HTML 200，但 `_next/static` JS/CSS 404，页面空白或打不开。
   - 防线：build 后继续预览时，停止旧 dev server，删除项目内 `.next`，重新 `npm run dev`。

8. 字体设置只改 CSS，不改首屏
   - 表现：设置页切换有效，刷新或其他页面无效。
   - 防线：根布局服务端读取 appearance，写入 `body data-font` 和 `--custom-font`。

## 每次修改设置系统后的最低验证

1. `npm run test:settings`
2. `npm run build`
3. GET `/api/settings` 不包含 mojibake 标记。
4. 对新增字段做 GET/PUT/GET 往返，验证保存和恢复。
5. `/settings` 返回 200。
6. 需要视觉生效的设置，要验证目标页面首屏也生效。
