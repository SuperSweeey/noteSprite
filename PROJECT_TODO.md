# NoteSprite 项目 To Do

本文件是长期任务总表。以后每次改动前先看这里：先判断新需求属于哪个层级，再更新状态，避免后面的需求覆盖前面的任务。

## 执行规则

- P0 优先于 P1/P2/P3。
- 新需求、测试报告问题、用户现场反馈必须先归类到本文件，写清优先级、验收标准和状态，再开始改代码；后续开发严格按本文件优先级顺序推进。
- 每发出一项功能后必须进行充分、准确、贴近真实使用的测试，覆盖成功路径、失败路径、重试路径、页面展示和关键数据状态；不能只看脚本全绿就视为稳定。
- 每次完成任务后，更新状态和验收结果。
- 每次收尾至少验证：`npm run build`、核心页面 200、关键 API。
